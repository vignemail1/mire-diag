Assistant de diagnostic de connexion

Cet outil vous guide pas à pas pour identifier et documenter votre problème de connexion.

Etape 1 — Informations générales

Aide VNC : Si vous utilisez une session (ex: :1), l'adresse est le nom du serveur et le port est 5900 + session (ex: session :1 = port 5901).

Etape 2 — Résolution DNS

Vérifiez que le nom du service se traduit bien en adresse IP. Lancez la commande ci-dessous.

Commande(s) à exécuter (IPv4) :
nslookup ... Copié !
Commande(s) à exécuter (IPv6) :
nslookup -type=AAAA ... Copié !

Etape 3 — Connectivité réseau (Ping)

Le ping vérifie si votre machine peut joindre le serveur.

Commande IPv4 :
ping ... Copié !
Commande IPv6 :
ping -6 ... Copié !

Etape 4 — Chemin réseau (Traceroute)

Le traceroute montre chaque étape du trajet entre vous et le serveur.

Commande IPv4 :
tracert ... Copié !
Commande IPv6 :
tracert -6 ... Copié !

Etape 5 — Test de connexion au service

Commande de test :
nc -zv ... Copié !
Test supplémentaire :
